01 — Overview

What Are We Building?

A smart system where users enter symptoms and get instant disease predictions — powered by ML and served through a modern MERN web app.

🎯

The Problem

People often don't know which doctor to visit. Early disease prediction can save lives and reduce unnecessary hospital visits.

💡

Our Solution

A beautiful React web app where users select symptoms → Node.js backend calls the Python ML model → instant disease prediction with confidence score.

🏆

Key Features

Multi-disease prediction, user authentication, prediction history, doctor recommendations, beautiful dashboard with charts.

🎯 Why MERN + Python?

MERN gives us a professional, scalable full-stack web app with user accounts and data storage.
Python is the industry standard for ML — Scikit-Learn, Pandas, NumPy are all Python libraries.
Together, they create a production-ready application that looks great on your portfolio.

🔑 Key Insight — Two Separate Servers

Server 1 (Node.js:5000) — Handles user auth, routes, talks to MongoDB and calls the ML service.

06 — ML Models

Models We'll Train & Compare

Try multiple models, compare accuracy, pick the winner.

ModelHow It Works (Simple)StrengthsExpected Accuracy
Decision Tree Makes yes/no decisions like a flowchart Easy to understand & visualize ~90-93%
Random Forest ⭐ Many trees vote together for the answer Very accurate, handles noise ~94-97%
SVM Draws the best boundary between disease groups Great for complex boundaries ~92-95%
Naive Bayes Uses probability — "what's most likely?" Fast, good with yes/no data ~88-92%
KNN Checks most similar past patients Simple concept, no training ~85-90%

⭐ Our Pick: Random Forest Classifier

Like asking 100 doctors (trees) and going with the majority opinion. It's robust, handles missing data, and rarely overfits. Usually gives ~95% accuracy for disease prediction datasets.

08 — Python ML Service (Flask)

Flask ML Microservice

A lightweight Python server that loads the trained model and serves predictions via a REST API.

🐍 Flask API (app.py)

from flask import Flask, request
import joblib, numpy as np

app = Flask(__name__)
model = joblib.load('models/model.pkl')
encoder = joblib.load('models/encoder.pkl')

@app.route('/predict', methods=['POST'])
def predict():
  data = request.json
  symptoms = data['symptoms']
  vector = create_vector(symptoms)
  prediction = model.predict(vector)
  proba = model.predict_proba(vector)
  return {
    'disease': encoder.inverse_transform(prediction)[0],
    'confidence': round(max(proba[0])*100, 2)
  }

🔄 How It Works

The Flask server is a separate mini-service. Node.js calls it internally — users never see it directly.

1 Flask loads model.pkl on startup (once)
2 Node.js sends POST request with symptoms list
3 Flask converts symptoms to number vector (0s and 1s)
4 Model predicts disease + confidence percentage
5 Returns JSON response back to Node.js

💡 Why Flask Instead of FastAPI?

Both work great! Flask is simpler and more beginner-friendly — just a few lines of code. FastAPI is faster and has auto-docs, but Flask is enough for this project. You can use either one — the concept is the same: a Python server that serves the ML model.
